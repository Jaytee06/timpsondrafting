# Service-Specific Qualification Follow-Up

## Node Type

Process subdomain.

## Purpose

This subtree holds communication guidance for project types that need more detailed follow-up than the shared qualification rules provide.

Use these nodes after the lead is clearly real and contactable, but before the lead should be treated as fully qualified for estimating or deeper sales handling.

## Current Child Nodes

- `addition.md`

## Routing Guidance

Stay in the parent qualification workflow when the missing information is generic.

Descend here when the service type is already known and the missing information depends on that service type.

Escalate upward if a new service-specific node would redefine shared lead fields or global qualification thresholds.
